# Carousel Frameworks

Use these frameworks to choose a narrative arc. Pick one primary framework per carousel unless the user asks for multiple versions.

## 1. How-To Framework

Best for tutorials, guides, and practical advice.

1. Hook: Promise a useful outcome.
2. Problem: Name what makes the outcome hard.
3. Principle: Explain the key idea.
4. Step 1: First action.
5. Step 2: Second action.
6. Step 3: Third action.
7. Example: Show it in practice.
8. CTA: Invite next action.

## 2. Mistakes Framework

Best for education, audits, and expert positioning.

1. Hook: Name the costly mistake pattern.
2. Stakes: Explain why it matters.
3. Mistake 1: Name and correct it.
4. Mistake 2: Name and correct it.
5. Mistake 3: Name and correct it.
6. Better model: Show the new approach.
7. Recap: Compress the fixes.
8. CTA: Ask readers to apply or respond.

## 3. Myth-Busting Framework

Best for contrarian posts and category education.

1. Hook: Challenge a common belief.
2. Myth: State the belief fairly.
3. Reality: Explain the better interpretation.
4. Evidence or example: Make the reframe believable.
5. Implication: Show what changes.
6. Action: Explain what to do instead.
7. Recap: Contrast myth versus reality.
8. CTA: Invite reflection or discussion.

## 4. Before-After Framework

Best for transformations, product education, and coaching.

1. Hook: Promise a visible transformation.
2. Before: Describe the current pain.
3. Hidden cause: Explain why the pain persists.
4. Shift: Introduce the new principle.
5. After: Describe the improved state.
6. Steps: Show how to get there.
7. Example: Make it concrete.
8. CTA: Invite the next step.

## 5. Case Study Framework

Best for proof, lessons learned, or project recaps.

1. Hook: State the result or lesson.
2. Context: Who or what the case involved.
3. Challenge: What made it difficult.
4. Action: What changed.
5. Result: What happened.
6. Lesson: What others can apply.
7. Playbook: Turn it into steps.
8. CTA: Invite readers to use the lesson.

Use only real case details provided by the user. Do not invent client names, metrics, quotes, or outcomes.

## 6. Product Story Framework

Best for launches, feature explainers, and product marketing.

1. Hook: Name the customer problem.
2. Old way: Show the friction.
3. Insight: Explain what the product changes.
4. Feature: Connect one feature to the problem.
5. Benefit: Translate feature into outcome.
6. Proof: Use provided proof or demo detail.
7. Use case: Show who it is for.
8. CTA: Invite trial, demo, signup, or reply.

## 7. Content Repurposing Framework

Best for turning long content into short social content.

1. Hook: Use the strongest takeaway from the source.
2. Setup: Explain what the source was about.
3. Big idea: State the main thesis.
4. Supporting point 1: Use the strongest argument.
5. Supporting point 2: Use the second strongest argument.
6. Supporting point 3: Use the most actionable point.
7. Summary: Give the distilled lesson.
8. CTA: Link the carousel to the next action.

## 8. Checklist Framework

Best for tactical execution.

1. Hook: Promise a checklist outcome.
2. Why it matters: Explain the stakes.
3. Checklist item 1.
4. Checklist item 2.
5. Checklist item 3.
6. Checklist item 4.
7. Quick self-audit.
8. CTA.

## Topic-only handling

When the user provides only a topic:

1. State assumptions briefly.
2. Avoid pretending to have source-specific evidence.
3. Use evergreen principles unless current research is available.
4. Make the carousel useful with examples, definitions, and steps.
5. Label any speculative or illustrative examples.
